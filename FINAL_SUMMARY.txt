================================================================================
                            KEY FEATURES
================================================================================

✅ Authorization
   - Only borrower/buyer can cancel their own request
   - Owner gets 403 Forbidden if they try
   - Enforced server-side

✅ Status Validation
   - Can only cancel in PENDING_NEGOTIATION or AWAITING_PAYMENT
   - Cannot cancel after payment confirmed (RESERVED+)
   - Clear error messages for each case

✅ Idempotency
   - Double-clicks don't deduct reputation twice
   - Network retries are safe
   - Guaranteed exactly -3 points per transaction
   - Implemented via cancelledAt timestamp

✅ Atomic Operations
   - MongoDB sessions wrap all steps
   - Transaction status update + reputation deduction + notification
   - All or nothing - no partial updates

✅ Owner Notification
   - Type: REQUEST_CANCELLED
   - Title: "Rental Request Cancelled" or "Purchase Request Cancelled"
   - Message includes borrower name and product name
   - Appears in notification panel immediately

✅ Reputation Integration
   - -3 points deducted from borrower/buyer
   - Appears in reputation history as "REQUEST_CANCELLED"
   - Visible in user profile
   - Exact once guarantee

✅ Transaction Preservation
   - Transaction not deleted
   - Status changed to CANCELLED_BY_BORROWER
   - Full history maintained
   - cancelledAt timestamp recorded

✅ UI/UX
   - Clear status badges
   - User-friendly messages
   - Dynamic button labels
   - Toast notifications
   - Chat remains available
   - Smooth state transitions

================================================================================
                            API ENDPOINT
================================================================================

Endpoint: POST /rent/transaction/:id/cancel
Auth: Required (Bearer token)
Payload: None

Success Response (200 OK):
{
  "msg": "Request cancelled successfully. -3 reputation deducted.",
  "transaction": {
    "_id": "...",
    "status": "CANCELLED_BY_BORROWER",
    "cancelledAt": "2026-07-03T12:34:56.789Z",
    ...
  }
}

Error Responses:
- 404 Not Found: Transaction doesn't exist
- 403 Forbidden: User is not the borrower
- 409 Conflict: Transaction not in cancellable state
- 200 OK: Transaction already cancelled (idempotent)
